# U2 实现说明（通义千问嵌入 + 向量 1024 维）

本文档说明 **升级策划书 U2**：将默认向量维度提升至 **1024**（与 `text-embedding-v3` 常用配置一致），新增 **`vagent.embedding.provider=dashscope`** 的 `EmbeddingClient` 实现，调用兼容模式 **`POST /v1/embeddings`**；保留 **`hash`** 供无网测试。

---

## 1. U2 要达成什么

| 项 | 说明 |
|----|------|
| DDL | `kb_chunks.embedding` 改为 **`vector(1024)`**；`KbChunkMapper` 中 `CAST(... AS vector(1024))` 同步。 |
| 配置 | `vagent.embedding.dimensions: 1024`；`vagent.embedding.dashscope.*`（`api-key`、`model` 等）。 |
| 实现 | `DashScopeEmbeddingClient` + `DashScopeEmbeddingsResponseParser`；请求体带 `dimensions` 与库表一致。 |
| 测试 | `HashEmbeddingClient` 与维度无关；单测 `dimensions=1024`；解析器单测无网。 |

---

## 2. 破坏性变更（已有库必看）

若库中已是 **`vector(128)`** 且存有数据：

1. **不能**直接把列改成 1024 而沿用旧向量（维度语义不兼容）。  
2. 常见做法：**删除 `kb_chunks`（或整库按 schema 重建）**，重新执行 `schema-vector.sql`，再 **重新入库** 文档。  

开发环境可 `DROP TABLE kb_chunks CASCADE;` 后重跑应用初始化 SQL（注意备份）。

---

## 3. 推荐阅读顺序

1. `schema-vector.sql`、`KbChunkMapper`、`EmbeddingProperties`。  
2. `EmbeddingClientConfiguration`、`DashScopeEmbeddingClient`、`EmbeddingDashScopeProperties`。  
3. `application.yml` 中 `vagent.embedding` 块。

---

## 4. 自测建议

- **无 Key**：`provider: hash`，`mvn test`、M2 Testcontainers 集成测试应通过。  
- **有 Key**：`provider: dashscope`，`POST /kb/documents` 入库后 `POST /kb/retrieve`，观察命中与距离合理。  
- **与 U1 联调**：聊天 `dashscope` + 嵌入 `dashscope`，共用 `DASHSCOPE_API_KEY`。

---

## 5. 与 U1、DECISIONS 的关系

- U1 负责 **流式聊天**；U2 负责 **检索用向量**；二者可独立选 `noop`/`hash` 与 `dashscope` 组合（开发时常用 **hash 嵌入 + dashscope 聊天** 省嵌入调用）。  
- [DECISIONS.md](DECISIONS.md) 中「真实 Embedding」在启用 `dashscope` 嵌入后缩小与 §3 的差异。

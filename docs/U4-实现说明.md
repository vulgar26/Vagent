# U4：可观测与工程化（traceId、Micrometer）

本文档说明 **升级策划书 U4** 在本仓库的落地方式。

## 1. 请求级 traceId（MDC）

| 项 | 说明 |
|----|------|
| 过滤器 | `com.vagent.observability.TraceIdMdcFilter`，置于 `JwtAuthenticationFilter` 之前 |
| MDC 键 | `traceId` |
| 入站头 | 优先 `X-Request-Id`，否则 `X-Trace-Id`；均无则生成 UUID |
| 出站头 | `X-Trace-Id`（与入站别名一致，便于前端读取） |
| 异步 | `llmStreamExecutor` 使用 `MdcTaskDecorator`，使 `llm-stream-*` 线程日志带同一 `traceId` |

日志格式见 `application.yml` 中 `logging.pattern.console`（含 `[%X{traceId:-}]`）。

## 2. Micrometer 指标

| 名称 | 类型 | 说明 |
|------|------|------|
| `vagent.rag.retrieve` | Timer | `KnowledgeRetrieveService#search` 全段（嵌入 + SQL） |
| `vagent.chat.stream` | Timer | `LlmSseStreamingBridge` 一次 `streamChatToSse` 从调用到流结束；**tag** `outcome`：`success` / `cancelled` / `error` |

**说明**：澄清 / U3 固定文案等**不经** `LlmSseStreamingBridge` 的路径**不计入** `vagent.chat.stream`。

Actuator 已暴露 `metrics`（见 `management.endpoints.web.exposure.include`），可 `GET /actuator/metrics/vagent.chat.stream` 等查看。

## 3. 关键阶段日志

`RagStreamChatService` 在拼好 `hits` / `systemText` 后打 **DEBUG**：`stream branch={} hitCount={}`（需将 `com.vagent.chat` 包日志级别设为 DEBUG 才可见）。

## 4. Flyway / Liquibase（生产建议）

开发期仍使用 `spring.sql.init.mode=always` 执行 `schema-*.sql`。**上线环境**建议改为：

- `spring.sql.init.mode=never`（或 `embedded` 仅内嵌库）
- 使用 **Flyway** 或 **Liquibase** 管理 DDL 版本；向量扩展 `CREATE EXTENSION vector` 通常需 DBA/超级用户预先执行一次。

本仓库 **未**引入 Flyway 依赖，避免与现有 init 行为冲突；迁移脚本拆分与 profile 切换可在后续迭代单独立项。

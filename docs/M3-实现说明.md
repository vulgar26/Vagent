# M3 实现说明（SSE 流式 LLM、任务取消）

本文档说明 **M3 里程碑**：在已有会话上通过 **Server-Sent Events** 将 `LlmClient#streamChat` 输出推到客户端，并提供 **按任务取消**；与 M0/M1/M2 的关系、配置与自测方式。

---

## 1. M3 要达成什么

| 目标 | 说明 |
|------|------|
| SSE | `POST /api/v1/conversations/{conversationId}/chat/stream`，`Content-Type: text/event-stream`，每条事件体为 **JSON**（`type` 区分 meta/chunk/done/cancelled/error）。 |
| 任务 id | 首条事件 `type=meta` 含 `taskId`，供取消接口使用。 |
| 取消 | `POST /api/v1/chat/tasks/{taskId}/cancel`，校验 **任务属主** 后标记取消；流式实现通过 `LlmChatRequest#isCancelled` 轮询退出。 |
| LLM 实现 | 保留 **`noop`**（无 chunk、立即结束）；新增 **`fake-stream`**（本地分块回显最后一条 USER 消息，可调分块大小与延迟）。 |
| 会话校验 | 仅允许对 **当前用户拥有** 的 `conversationId` 发起流式请求（与 M1 数据隔离一致）。 |

**M3 刻意不包含**：RAG 编排、历史消息持久化、真实厂商流式 HTTP、SSE 断线重连与幂等。

---

## 2. 与 M0/M1/M2 的关系

- **M0**：`LlmClient` / `LlmStreamSink` 契约不变；M3 将 sink 桥接到 `SseEmitter`。  
- **M1**：JWT + `VagentUserPrincipal`；`ConversationService#findOwnedByUser` 校验会话归属。  
- **M2**：独立；知识库检索未接入流式对话（后续里程碑编排）。  
- **线程**：流式逻辑在 **`llmStreamExecutor`** 线程池执行，避免阻塞 Servlet 线程。

## 4. SSE 事件格式（JSON）

| type | 含义 |
|------|------|
| `meta` | 含 `taskId` |
| `chunk` | 含 `text` |
| `done` | 正常结束 |
| `cancelled` | 用户取消后结束 |
| `error` | 含 `message` |

客户端推荐使用 **`fetch` + `ReadableStream`** 携带 `Authorization: Bearer`；浏览器原生 **EventSource** 无法自定义请求头，生产环境需另行设计（Cookie、网关注入等）。

---

## 5. 配置

| 配置项 | 说明 |
|--------|------|
| `vagent.llm.provider` | `noop`（默认）或 `fake-stream`。 |
| `vagent.llm.fake-stream-chunk-size` | 每段字符数（≥1）。 |
| `vagent.llm.fake-stream-chunk-delay-ms` | 段间延迟（毫秒），0 表示不 sleep。 |

## 6. 测试

- `M3StreamChatIntegrationTest`：无会话 id 流式 → 404；未知 `taskId` 取消 → 404。  
- `spring.mvc.async.request-timeout` 在 `application-test.yml` 中加长，避免异步 SSE 测试超时。

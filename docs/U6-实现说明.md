# U6：MCP（独立进程，HTTP + Python）

本文档说明 **升级策划书 U6** 在本仓库的最小落地：Vagent 仅作为 **MCP Client**，通过 HTTP 调用独立的 Python MCP Server。

## 1. 形态与边界

- **Server**：独立 Python 进程（建议单独仓库维护），对外提供 MCP **Streamable HTTP** 传输。  
- **Client**：Vagent 内 `HttpMcpClient`；当前采用 **JSON-only 响应模式**（不解析 SSE 流）。

> 提示：Python 侧需启用 streamable-http 的 JSON 响应（Python SDK transport 支持 `is_json_response_enabled`）。

## 2. 配置

| 键 | 说明 |
|----|------|
| `vagent.mcp.enabled` | 是否启用 MCP Client（默认 `false`） |
| `vagent.mcp.base-url` | MCP Server URL（需指向 Streamable HTTP 路径），例如 `http://127.0.0.1:8765/mcp` |
| `vagent.mcp.token` | Bearer token（建议环境变量 `MCP_TOKEN` 注入） |
| `vagent.mcp.protocol-version` | MCP 协议版本（默认 `2025-03-26`） |
| `vagent.mcp.allowed-tools` | （U7）主链路允许调用的工具白名单（逗号分隔）；为空则主链路不允许任何工具 |

## 3. 联调 API（需登录）

启用 `vagent.mcp.enabled=true` 后：

- `GET /api/v1/mcp/tools`：列出工具（`tools/list`）
- `POST /api/v1/mcp/tools/{name}`：调用工具（`tools/call`），body 为 `arguments`（JSON object）

## 4. 可观测

- 日志：沿用 U4 `traceId`（MDC）  
- 指标：`vagent.mcp.call`（Timer，tags：`tool`、`outcome`），以及初始化通知的 `vagent.mcp.notify`

## 5. 备注：主链路集成（U7）

U6 在本仓库的落地以“联调入口”为主：`/api/v1/mcp/*` 用于验证工具列表与调用是否打通。  
后续若将 MCP 工具并入 RAG 主链路（对齐策划书 §3 的“工具侧上下文”），建议采用：

- **显式触发**：由规则/意图决定是否允许调用工具（默认不调用），避免 prompt 注入；
- **白名单**：仅允许 `vagent.mcp.allowed-tools` 中列出的工具进入主链路；
- **超时与降级**：工具失败时继续走 KB RAG（或无工具继续回答），并记录指标与日志。

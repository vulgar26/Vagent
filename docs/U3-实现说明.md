# U3：空检索行为（empty-hits-behavior）

本文档说明 **升级策划书 U3**：当 **RAG 分支**完成向量检索且 **命中数为 0** 时，是否仍调用 `LlmClient`。

## 配置

| 键 | 说明 |
|----|------|
| `vagent.rag.empty-hits-behavior` | `no-llm`：不调 LLM，推送固定文案后经 SSE `done`；`allow-llm`：**默认**，与旧版一致（SYSTEM 说明未命中后仍调 LLM）。 |
| `vagent.rag.empty-hits-no-llm-message` | 可选；`no-llm` 时推送的固定全文，覆盖内置默认文案。 |

## 行为边界

- 仅作用于 **`ChatBranch.RAG`** 且已执行检索、**`hits` 为空**的路径。  
- **寒暄**（`SYSTEM_DIALOG`）不经检索，不适用本开关。  
- **澄清**（`CLARIFICATION`）本来就不调主链路 LLM，与本配置无关。

## SSE

与澄清分支一致：`meta`（`hitCount=0`，`branch=RAG`）→ `chunk` → `done`，并落库 ASSISTANT 消息。

## 与策划书 §3 / DECISIONS

`no-llm` 时对齐「检索为空则不调大模型」；`allow-llm` 保留 DECISIONS 中描述的「仍调 LLM」历史行为。

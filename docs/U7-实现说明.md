# U7：MCP 工具并入主链路（显式触发 + 参数白名单）

本文档说明 **U7** 在本仓库的落地：在 RAG 主链路中按“显式工具意图”触发 MCP 工具调用，并将工具结果并入 Prompt 上下文。

## 1. 目标与边界

- **目标**：让 MCP 从“联调入口（/api/v1/mcp/*）”升级为可用于 **主对话链路** 的能力。
- **安全边界**：
  - 默认不调用工具（需显式开启）。
  - 只允许 **白名单** 工具进入主链路。
  - 工具参数在服务端做**最小白名单收敛**，避免透传任意参数。
- **当前形态**：Vagent 仍作为 MCP Client（HTTP，JSON-only），MCP Server 为独立进程（Python）。

## 2. 配置项

### 2.1 启用 MCP Client（U6）

| 键 | 说明 |
|----|------|
| `vagent.mcp.enabled` | 是否启用 MCP Client（默认 `false`） |
| `vagent.mcp.base-url` | MCP Server URL（含 `/mcp`），例如 `http://127.0.0.1:8765/mcp` |
| `vagent.mcp.token` | Bearer token（建议环境变量 `MCP_TOKEN` 注入） |
| `vagent.mcp.allowed-tools` | 主链路允许调用的工具白名单（逗号分隔），例如 `echo,ping` |
| `vagent.mcp.tool-call-timeout` | 单次 `tools/call` 超时（主链路），例如 `PT3S` |

### 2.2 启用“工具意图”（U7）

| 键 | 说明 |
|----|------|
| `vagent.orchestration.tool-intent-enabled` | 是否启用工具意图（默认 `false`） |
| `vagent.orchestration.tool-intent-default-tool-name` | 关键词兜底时默认工具名（例如 `echo`） |
| `vagent.orchestration.tool-intent-keywords` | 关键词兜底（推荐仍以显式指令触发为主） |

## 3. 触发语法（显式）

当 `tool-intent-enabled=true` 时，用户输入满足以下格式可触发工具意图：

- `tool=echo message=hello`
- `tool=echo message="hello world"`
- `/tool ping`
- `/tool echo message='你好 世界'`

其中：

- `NAME` 必须在 `vagent.mcp.allowed-tools` 白名单中，否则主链路会跳过工具调用。
- 当前内置参数白名单收敛（示例）：
  - `ping`：不接受参数
  - `echo`：仅允许 `message`

## 4. 主链路行为

在 `RagStreamChatService` 的 RAG 分支中：

1. 若检测到工具意图且工具在白名单中，则调用 MCP `tools/call` 获取工具结果。
2. 工具结果会被拼入 system prompt（作为“工具上下文”），再与 KB 片段一起送入 LLM。
3. SSE `meta` 会附加：
   - `toolUsed`: 是否成功使用了工具上下文
   - `toolName`: 工具名（当 `toolUsed=true`）
   - `toolOutcome`: `success` / `timeout` / `error`（当发生工具调用尝试时）
   - `toolError`: 错误信息（当 `toolOutcome!=success`）

## 5. 本地启动与演示（Windows）

### 5.1 启动 Python MCP Server

在 `D:\Projects\PythonMCPServer`：

```powershell
Set-Location D:\Projects\PythonMCPServer
$env:MCP_TOKEN="testsecret"
.\.venv\Scripts\python -m uvicorn server:build_starlette_app --factory --host 127.0.0.1 --port 8765
```

### 5.2 启动 Vagent（示例）

确保环境变量与配置一致：

```powershell
$env:MCP_TOKEN="testsecret"
```

并在 `application.yml`（或本地 profile）中设置：

- `vagent.mcp.enabled: true`
- `vagent.mcp.base-url: http://127.0.0.1:8765/mcp`
- `vagent.orchestration.tool-intent-enabled: true`
- `vagent.mcp.allowed-tools: echo,ping`

启动后，在对话中输入：

- `tool=echo message="hello world"`

观察 SSE `meta` 中的 `toolUsed/toolName/toolOutcome`。

## 6. 常见问题

- **报错“未配置 MCP token”**：`vagent.mcp.enabled=true` 时必须配置 `vagent.mcp.token`（推荐用环境变量 `MCP_TOKEN`）。
- **工具不触发**：检查 `tool-intent-enabled`、输入格式是否正确、以及 `allowed-tools` 白名单是否包含目标工具。
- **超时**：可调大 `vagent.mcp.tool-call-timeout`，或检查 MCP Server 是否正常启动与可达。

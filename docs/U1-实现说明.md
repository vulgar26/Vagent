# U1 实现说明（通义千问 DashScope 兼容模式流式）

本文档说明 **升级策划书 U1**：在保留 `noop` / `fake-stream` 的前提下，新增 **`vagent.llm.provider=dashscope`**，通过 **OpenAI 兼容 Chat Completions** 流式接口调用阿里云通义千问，与现有 `LlmClient` / `LlmSseStreamingBridge` 全链路对接。

---

## 1. U1 要达成什么

| 目标 | 说明 |
|------|------|
| 真实流式 | `DashScopeCompatibleStreamingLlmClient`：`HttpClient` POST `.../chat/completions`，`stream:true`，按 SSE `data:` 行解析增量。 |
| 配置 | `vagent.llm.dashscope.*`：`base-url`、`api-key`（建议 `${DASHSCOPE_API_KEY:}`）、`chat-model`。 |
| 安全 | API Key 不入库、不写死进仓库；默认 `application.yml` 仍为 `provider: noop`。 |
| 测试 | CI 无网：`application-test.yml` 固定 `provider: noop`；解析逻辑见 `DashScopeOpenAiStreamParserTest`。 |

## 4. 自测建议（需有效 API Key）

1. 环境变量：`DASHSCOPE_API_KEY=sk-...`  
2. `application.yml`：`vagent.llm.provider: dashscope`（或本地 profile 覆盖）。  
3. 启动 PG + 应用，`fake-stream` 非必须；走 `POST .../chat/stream`，观察 SSE `chunk` 为真实模型输出。  
4. 验证 `POST .../chat/tasks/{taskId}/cancel` 在流式过程中仍可触发停止（与 M3 语义一致，厂商侧是否立即停止计费以云文档为准）。
